Adding an Item

This guide assumes you have a modding workspace setup.

1. Setup Registry Event

You need to register your item to the game, but to do this you need to hook into the register items event. To do this you should subscribe to the event net.minecraftforge.event.RegistryEvent.Register<Item>.

Learn more about subscribing to events

2. Registering a basic item

To register an item you need to get the registry from the event using event.getRegistry() and use the .register or .registerAll method. To add a basic item, instantiate the item class, and pass it a new `Item.Properties`. See and example below.

event.getRegistry().register(new Item(new Item.Properties().group(ItemGroup.MISC)).setRegistryName("mymodid:myitemname"));

Note: DO NOT USE setRegistryName in the Item's constructor. This is a fast way to make your mod unexpandable, i.e. noone will be able to use your item class if they want to create an addon which expand on it's functionality.

3. Using the new item

If you need to reference this item somewhere in code it would be useful to have an instance of it available for you to work with right. This is where you should use @ObjectHolder. This annotation will inject a valid registry value into the field it is annotating. See how to use it in the example below.

@ObjectHolder("mymodid:myitemname") public static final Item MY_ITEM_NAME = null;

Note: Using @ObjectHolder is important for you to use if you want your mod to be friendly to work with.

4. Add an item model and lang entry

This item will need a model in resources/assets/mymodid/models/item/myitemname.json. Furthermore you will need a lang entry in your en_us.json in the format "item.mymodid.myitemname"="Item Name".
